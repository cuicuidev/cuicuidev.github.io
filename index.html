My Work

My Gym Bro

This is an idea I had a while ago. I'm a regular gym goer and I thought it would be convinient to have a mobile app that allowed me to track my progress and visualize it. This is exactly that.

I am using Django as my back-end, PostgreSQL as my database and Gunicorn along with Nginx as my server. Everything is containerized with Docker. This API allows crud operations on gym tracking entries and JSON Web Token authentication methods with RS512 encription. There is also CI/CD pipelines in place that check dev and production environments setup.

The front-end is not yet created, although I'm planning on using Flutter for the mobile app and tackle with a new JS framework to showcase the product on the Internet.

I've got to admit, this is my most ambitious project so far and not even the back-end is ready for deployment. I fear I might have taken a task too big for one person to acomplish. However, I want to take on the challenge and learn from the experience.

Even if this is an unfinished project, I want to showcase it here regardless, so that you can always take a sneak peek and see what I am capable of!
